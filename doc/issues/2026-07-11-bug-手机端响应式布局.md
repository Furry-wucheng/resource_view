# bug: 手机端响应式布局与查看行为异常

> 日期: 2026-07-11 | 类型: bug | 状态: ✅ 已完成

## 现象

1. 手机进入文件浏览器后目录树默认展开，并以内联分栏挤压内容区。
2. 手机 SMB 添加/编辑弹窗仍使用平板双列表单，输入框过窄。
3. 手机数据源卡片操作按钮拥挤，资源拆分弹窗的标题和长按钮可能溢出。
4. 手机横屏时查看器自动启用双页，与仅宽屏启用双页的产品约定不一致。

## 复现步骤

1. 在宽度小于 900dp 的手机上进入任意数据源的文件浏览器。
2. 打开 SMB 添加或编辑弹窗，观察端口/域名和用户名/密码字段。
3. 查看 SMB 数据源卡片，并打开资源拆分弹窗。
4. 将手机旋转到横屏，使用自动或双页查看模式打开图片资源。

## 期望效果

- 窄屏目录树默认关闭，通过菜单打开；目录跳转后关闭，展开/折叠操作不关闭。
- 手机 SMB 字段各占一行；宽屏保持双列。
- 手机卡片和弹窗操作不拥挤、不溢出。
- 宽度小于 900dp 时始终单页，宽屏才允许双页。

## 影响分析

| 维度 | 内容 |
|------|------|
| 根因 | 文件浏览器和表单直接使用固定横向布局；部分弹窗/卡片未按可用宽度切换；双页判定只检查横竖屏 |
| 修改文件 | 文件浏览器、SMB 弹窗、数据源卡片、ResourcePicker、查看器及响应式测试 |
| 影响 stage | 修正 M04 响应式布局及文件浏览器、SMB、查看器相关既有产物，不新增 stage |
