# feature: 文件浏览器体验优化

> 日期: 2026-06-28 | 类型: feature | 状态: ✅ 已完成

## 描述

对源文件浏览器进行多项体验优化，包括：
1. 默认视图模式改为网格，并持久化用户选择
2. 支持多种排序方式（文件名正序/倒序、修改时间从旧到新/从新到旧），每层目录独立持久化排序
3. 改进导航栏为真正的树状目录结构，支持懒加载子目录
4. 调整顶栏按钮顺序：目录树 | 网格/列表 | 排序选择
5. 无缩略图时不显示右下角文件夹角标
6. 目录树在 content 内部，由按钮控制显示/隐藏
7. 返回键恢复上一层目录功能
8. 恢复面包屑导航
9. 设置页面添加目录树开关

## 验收标准

- [x] 默认视图模式为网格
- [x] 用户切换视图模式后，下次进入同一数据源时保持上次选择
- [x] 排序支持：文件名A-Z、文件名Z-A、修改时间从新到旧、修改时间从旧到新
- [x] 每层目录独立记住排序偏好
- [x] 导航栏显示真正的目录树结构，支持展开/折叠子目录
- [x] 顶栏按钮顺序为：目录树 | 网格/列表 | 排序选择
- [x] 目录树在 content 内部，由按钮控制
- [x] 返回键：多选退出 → 上一层目录 → 页面返回
- [x] 面包屑导航显示路径层级，可点击跳转
- [x] 无缩略图的文件不显示右下角文件夹角标
- [x] 设置页面可开关目录树功能

## 执行计划

### Phase 1: 数据持久化层
1. 新增 `FileBrowserPrefsStore` 使用 DataStore 存储文件夹偏好
2. 定义 `FileSortMode` 枚举和 `FolderPrefs` 数据模型
3. 最多存储 1000 个目录的偏好（LRU 淘汰）

### Phase 2: ViewModel 逻辑重构
1. 重构 `FileBrowserViewModel` 添加排序和视图模式逻辑
2. 实现 `setSortMode` 和 `setViewMode` 方法
3. 实现偏好加载和保存

### Phase 3: UI 重构
1. 新增 `DirectoryTree` 组件，实现扁平化列表 + 展开状态映射
2. 重构 `FileBrowserScreen`，目录树在 content 内部
3. 实现排序下拉菜单
4. 恢复面包屑导航
5. 优化网格项角标显示逻辑

### Phase 4: 设置页面
1. 在 `AppConfigEntity` 添加 `showDirectoryTree` 字段
2. 添加数据库迁移 4→5
3. 在设置页面添加目录树开关
