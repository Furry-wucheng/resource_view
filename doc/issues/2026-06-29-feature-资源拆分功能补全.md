# feature: 资源拆分功能补全 — UI 集成与交互闭环

> 日期: 2026-06-29 | 类型: feature | 状态: ✅ 已完成

## 描述

M27 完成了 `SplitResourceUseCase`（domain 层骨架）和 `ResourcePickerDialog`（UI 组件），但两者未连接：
- 首页长按资源缺少"拆分资源"上下文菜单
- ResourcePickerDialog 标题/按钮硬编码为"批量添加"模式
- HomeViewModel 未注入 SplitResourceUseCase
- 文件浏览器已入库目录缺少"拆分资源"入口
- SplitResourceUseCase 未自动检测子资源组织模式
- 拆分后父资源标记逻辑未区分"保留/删除"两种模式
- ResourceDao 未过滤 isAvailable，拆分后父资源仍可见

参照 Flutter 版完整链路和 PRD `@prd/01-资源库首页.md` §4.5、`@prd/07-资源组织结构.md` §5.6 补全。

## 验收标准

- [x] HomeScreen 长按资源弹出上下文菜单，含"拆分资源"选项
- [x] ResourcePickerDialog 支持 splitKeep/splitDelete 模式，标题/按钮正确
- [x] HomeViewModel 编排完整拆分流程（Picker → 确认 → 标签(可选) → 执行）
- [x] 拆分后自动检测子资源组织模式（调用 DetectOrganizationModeUseCase）
- [x] 拆分支持"保留原资源"和"删除原资源"两种模式
- [x] 拆分后父资源标记 isAvailable=false，首页不再显示
- [x] FileBrowserScreen 已入库目录右键菜单含"拆分资源"
- [x] ResourceDetailSheet 含"拆分资源"操作
- [x] 拆分子资源基于 sourceId+relativePath 去重
- [x] ./gradlew build + test 通过
