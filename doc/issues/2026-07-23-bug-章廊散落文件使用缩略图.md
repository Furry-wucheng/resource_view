# bug: 章廊散落文件固定显示文件夹图标

> 日期: 2026-07-23 | 类型: bug | 状态: ✅ 已完成

## 现象

章廊模式中的散落图片、视频和 PDF 均显示固定文件夹图标，没有像章节封面和平铺模式一样加载内容缩略图。

## 复现步骤

1. 打开根目录中含有散落媒体文件的章廊资源。
2. 查看列表或网格视图的“散落文件”区域。
3. 所有文件均显示固定文件夹图标。

## 期望效果

散落文件通过现有缩略图加载管理器展示缓存缩略图；加载失败时才显示回退图标。

## 影响分析

| 维度 | 内容 |
|------|------|
| 根因 | `LooseFileItem` 与 `LooseFileGridItem` 未接入 `ThumbnailLoadManager`，直接硬编码 `Folder` 图标 |
| 修改文件 | `ChapterListViewModel.kt`、`ChapterListScreen.kt` 及 ViewModel 测试 |
| 影响 stage | M21 章节策略 UI、M23 缩略图能力复用 |

## 执行计划

1. RED：验证散落文件缩略图委托给共享加载管理器。
2. GREEN：ViewModel 新增散落文件缩略图入口。
3. UI：列表显示裁剪缩略图，网格复用 `FileThumbnailCard`。
4. 运行定向测试及全量 test/build/lint。

## 产出

| 文件 | 操作 | 说明 |
|------|------|------|
| `ui/screens/viewer/ChapterListViewModel.kt` | 修改 | 新增散落文件共享缩略图加载入口 |
| `ui/screens/viewer/ChapterListScreen.kt` | 修改 | 列表显示裁剪缩略图，网格复用平铺缩略图卡片 |
| `ui/screens/viewer/ChapterListViewModelTest.kt` | 新增 | 验证缩略图委托与缓存策略参数 |

验证：定向测试先 RED 后 GREEN；`./gradlew test build lint` 全部通过。
