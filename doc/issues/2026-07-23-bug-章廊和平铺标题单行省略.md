# bug: 章廊和平铺标题未按章节模式单行省略

> 日期: 2026-07-23 | 类型: bug | 状态: ✅ 已完成

## 现象

章节模式中的章节标题限制为一行并以省略号截断，但章廊模式的章节卡片、散落文件卡片和平铺模式文件卡片允许显示两行，长标题表现不一致。

## 复现步骤

1. 打开包含长章节名或长文件名的资源。
2. 分别切换到章节、章廊和平铺模式。
3. 观察章廊和平铺标题占用两行，而章节模式在第一行末尾显示省略号。

## 期望效果

章廊和平铺模式的卡片标题与章节模式一致：最多显示一行，超出部分使用省略号隐藏。

## 影响分析

| 维度 | 内容 |
|------|------|
| 根因 | `ChapterListScreen` 的章廊卡片和 `FileThumbnailCard` 的平铺卡片将 `maxLines` 设置为 2 |
| 修改文件 | `FileThumbnailCard.kt`、`ChapterListScreen.kt`、标题策略测试 |
| 影响 stage | M21 章节策略、M20 内容网格 UI |

## 执行计划

1. RED：新增网格卡片标题最大行数策略测试。
2. GREEN：将章廊章节、章廊散落文件和平铺文件标题统一使用单行策略。
3. 运行定向测试及全量 test/build/lint。

## 产出

| 文件 | 操作 | 说明 |
|------|------|------|
| `ui/components/FileThumbnailCard.kt` | 修改 | 平铺卡片标题统一限制为一行 |
| `ui/screens/viewer/ChapterListScreen.kt` | 修改 | 章廊章节与散落文件标题统一限制为一行 |
| `ui/components/GridCardTitlePolicyTest.kt` | 新增 | 验证网格卡片采用单行省略策略 |

验证：定向测试先 RED 后 GREEN；`./gradlew test build lint` 全部通过。
