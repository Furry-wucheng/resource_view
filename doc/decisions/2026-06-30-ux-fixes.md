# ux-batch-fixes — 五项UX缺陷修复

> 日期: 2026-06-30 | Agent: opencode | 状态: ✅ 已完成 | 前置: 多 stage (M23/M24/M21/M14/M27)

## 设计决策

### D-001: 批量标签采用独立 BatchTagDialog（不复用 BatchAddResourcesDialog）
- **背景**: 首页多选模式下对已有资源批量添加标签，不需要组织模式选择
- **选择**: 创建轻量级 `BatchTagDialog`，仅包含标签勾选列表 + 新建标签输入框，不带组织模式 chips
- **备选**: 复用 `BatchAddResourcesDialog` 但需增加 `showOrgMode=false` 分支 → 组件职责混淆
- **影响文件**: `ui/screens/home/HomeScreen.kt` (私有 Composable)
- **被依赖**: HomeScreen 多选模式底部栏 → 按钮触发弹窗

### D-002: 跨章节导航采用 reloadChapterContent 实际重建 ContentProvider
- **背景**: 原有 `navigateToNextChapter()` 仅更新 `_currentChapterIndex` + 显示提示，不加载内容
- **选择**: 新增 `reloadChapterContent(chapterPath, startPage)` 方法：释放旧 ContentProvider → 重建 MixedFolderProvider → 更新 PageLoader + _uiState。上一章跳末页(startPage=Int.MAX_VALUE)，下一章跳首页(startPage=0)
- **备选**: 修改 contentPath 参数后重调 `loadResource()` — 会重建多处状态，且破坏已有资源信息缓存
- **影响文件**: `ui/screens/viewer/ViewerViewModel.kt:800-880`
- **注意**: 仅在 `_chapters` 非空时生效（隐含限定 CHAPTER/CHAPTER_GALLERY 模式）

### D-003: 已入库节点复选框隐藏策略
- **背景**: ResourcePickerDialog 中 `isImported=true` 的节点同时显示绿色勾号 badge 和复选框，视觉冗余且已入库不可重新入库
- **选择**: `isImported` 时仅显示绿色勾号 badge，隐藏复选框 Icon。clickable 自然失效
- **影响文件**: `ui/components/ResourcePickerDialog.kt:168-183`

### D-004: 散落文件统一走 FileViewer 路由
- **背景**: 散落文件在 ChapterListScreen 中点击后走 ChapterViewer 路由，导致 `MixedFolderProvider` 以文件路径为目录参数，同级其他文件不可见
- **选择**: 所有散落文件（图片/视频/PDF）统一走 `onOpenVideo` → FileViewer 路由，`ViewerScreen(sourceId, filePath)` 内部以父目录创建 MixedFolderProvider，天然支持同目录翻页
- **影响文件**: `ui/screens/viewer/ChapterListScreen.kt:370-380, 415-425`

### D-005: 详情弹窗删除确认采用 AlertDialog 嵌套
- **背景**: ResourceDetailSheet 是 ModalBottomSheet，删除操作需二次确认防误触
- **选择**: 在 BottomSheet 内容中管理 `showDeleteConfirm` 状态，点击"删除资源"按钮后弹出 AlertDialog 二次确认
- **影响文件**: `ui/components/ResourceDetailSheet.kt:205-245`
