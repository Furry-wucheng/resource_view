# 05 — 错误处理策略

> 日期：2026-06-25 · 状态：已定稿 · 依据：`@tech/02-架构设计.md`

---

## 1. 概述

定义从 DataSource → Repository → UseCase → ViewModel → Compose 的**统一错误传播约定**。

核心原则：**预期错误用 Result 包装，编程错误用异常（throw）。**

---

## 2. 错误分类

### 2.1 来源域

| 域 | 来源 | 典型场景 |
|----|------|---------|
| I/O | 文件系统、SMB 网络 | 路径不存在、断网、权限不足、超时 |
| 数据 | Room 数据库 | 约束冲突、迁移失败、磁盘满 |
| 媒体 | 图片解码、PDF 渲染、压缩包 | 损坏文件、不支持的格式、加密 PDF |
| 用户输入 | 表单校验 | 标签名空/重复/与内置冲突 |
| 状态 | 业务状态 | 资源不可达 (isAvailable=false) |

### 2.2 严重级别

| 级别 | 定义 | 策略 | 示例 |
|------|------|------|------|
| **Fatal** | 数据库损坏 | 全屏阻塞 | 数据库打开失败 |
| **Recoverable** | SMB 超时、图片加载失败 | 降级 + 重试 | 降级 UI + 用户重试 |
| **Expected** | 标签名重复 | Toast / Snackbar | 不打断流程 |

## 3. Result\<T\> 定义

```kotlin
sealed class Result<out T> {
    data class Ok<T>(val value: T) : Result<T>()
    data class Err(val error: DomainError) : Result<Nothing>()
}

sealed class DomainError(val message: String, val cause: Throwable? = null) {
    // I/O 类
    class SourceUnreachableError(message: String, cause: Throwable? = null) : DomainError(message, cause)
    class SourceAuthError(message: String, cause: Throwable? = null) : DomainError(message, cause)
    class FileNotFoundError(message: String, cause: Throwable? = null) : DomainError(message, cause)
    class FileAccessDeniedError(message: String, cause: Throwable? = null) : DomainError(message, cause)
    class NetworkTimeoutError(message: String, cause: Throwable? = null) : DomainError(message, cause)

    // 媒体类
    class MediaLoadError(val mediaType: MediaType, message: String, cause: Throwable? = null) : DomainError(message, cause)
    class MediaEncryptedError(val mediaType: MediaType, message: String, cause: Throwable? = null) : DomainError(message, cause)
    class MediaStreamError(val mediaType: MediaType, message: String, cause: Throwable? = null) : DomainError(message, cause)

    // 校验类
    class ValidationError(message: String) : DomainError(message)

    // 状态类
    class OperationCancelledError(message: String = "操作已取消") : DomainError(message)
    class InsufficientStorageError(message: String, cause: Throwable? = null) : DomainError(message, cause)
    class UnsupportedFormatError(val format: String, message: String) : DomainError(message)

    // 数据库类
    class DatabaseError(message: String, cause: Throwable? = null) : DomainError(message, cause)
}

enum class MediaType { IMAGE, PDF, ARCHIVE, VIDEO }
```

## 4. 各层约定

### 4.1 DataSource 层：只抛出，不捕获

```kotlin
class SmbDataSource(private val client: SmbClientWrapper) {
    // 直接抛 SMBApiException，不做域转换
    suspend fun listDirectory(path: String): List<SmbEntry> {
        return client.listDirectory(path)
    }
}
```

### 4.2 Repository 层：捕获 + 转为 Result\<T\>

```kotlin
class FilesystemRepository(private val factory: FileSourceFactory) {
    suspend fun listDirectory(sourceId: String, path: String): Result<List<FileEntry>> {
        return try {
            val source = factory.create(sourceId)
            val entries = source.listDirectory(path)
            Result.Ok(entries)
        } catch (e: SMBApiException) {
            when {
                e.status == NtStatus.STATUS_LOGON_FAILURE ->
                    Result.Err(DomainError.SourceAuthError("SMB 认证失败", e))
                else ->
                    Result.Err(DomainError.SourceUnreachableError("SMB 源不可达", e))
            }
        } catch (e: java.io.FileNotFoundException) {
            Result.Err(DomainError.FileNotFoundError("路径不存在: $path", e))
        } catch (e: java.io.IOException) {
            Result.Err(DomainError.FileAccessDeniedError("权限不足: $path", e))
        } catch (e: Exception) {
            Result.Err(DomainError.DatabaseError("未知错误: ${e.message}", e))
        }
    }
}
```

### 4.3 ViewModel 层：Result → UiState

```kotlin
enum class UiState { IDLE, LOADING, SUCCESS, ERROR }

abstract class BaseViewModel : ViewModel() {
    var uiState by mutableStateOf(UiState.IDLE)
        protected set
    var errorMessage by mutableStateOf<String?>(null)
        protected set
    var lastError by mutableStateOf<DomainError?>(null)
        protected set

    val canRetry: Boolean
        get() = lastError is DomainError.SourceUnreachableError ||
                lastError is DomainError.FileNotFoundError ||
                lastError is DomainError.NetworkTimeoutError ||
                lastError is DomainError.MediaLoadError

    protected fun <T> handleResult(result: Result<T>) {
        when (result) {
            is Result.Ok -> {
                uiState = UiState.SUCCESS
                errorMessage = null
            }
            is Result.Err -> {
                uiState = UiState.ERROR
                errorMessage = mapError(result.error)
                lastError = result.error
            }
        }
    }

    private fun mapError(error: DomainError): String = when (error) {
        is DomainError.SourceUnreachableError -> "数据源不可达，请检查网络连接"
        is DomainError.SourceAuthError -> "SMB 认证失败，请检查用户名和密码"
        is DomainError.FileNotFoundError -> "路径不存在或已被移除"
        is DomainError.NetworkTimeoutError -> "连接超时，请稍后重试"
        is DomainError.MediaLoadError -> when (error.mediaType) {
            MediaType.PDF -> "PDF 加载失败"
            MediaType.VIDEO -> "视频加载失败"
            MediaType.ARCHIVE -> "压缩包读取失败"
            MediaType.IMAGE -> "图片加载失败"
        }
        is DomainError.ValidationError -> error.message
        else -> "未知错误"
    }
}
```

### 4.4 Compose 层：按状态渲染

```kotlin
@Composable
fun FileBrowserContent(viewModel: FileBrowserViewModel) {
    when (viewModel.uiState) {
        UiState.IDLE -> { /* 空白 */ }
        UiState.LOADING -> FileBrowserSkeleton()
        UiState.ERROR -> ErrorView(
            message = viewModel.errorMessage ?: "",
            canRetry = viewModel.canRetry,
            onRetry = { viewModel.retry() },
        )
        UiState.SUCCESS -> FileListContent(viewModel.files)
    }
}
```

## 5. 特殊场景

### 5.1 源不可达

```
SMB listDirectory 抛异常
  → Repository 返回 Err(SourceUnreachableError)
  → ViewModel: sourceRepository.markUnavailable(sourceId)
  → 该源所有 Resource.isAvailable = false
  → 首页：该源资源置灰 + 标记
  → 用户手动同步 → markAvailable
```

### 5.2 图片/PDF/视频加载失败

```
ContentProvider.loadPage(index) 或 ExoPlayer 抛异常
  → ViewModel: pageErrors[index] = MediaLoadError
  → 当前页显示"加载失败"占位 + 重试按钮
  → 用户翻到其他页：正常加载
```

### 5.3 扫描入库

```
协程遍历文件
  → 每项：try → 成功入 successList / 失败入 failureList
  → 返回 ScanResult(successCount, skipCount, failures)
  → UI: Snackbar "成功导入 8 个资源，2 个跳过，1 个失败"
```

### 5.4 标签名校验

```kotlin
fun validateTagName(name: String, existingNames: List<String>): String? {
    if (name.isBlank()) return "标签名不能为空"
    if (name == "收藏") return "'收藏'是系统内置标签，请换一个名称"
    if (name in existingNames) return "标签'$name'已存在"
    if (name.length > 20) return "标签名不能超过 20 个字符"
    return null
}
```

## 6. 流式操作（Progress）

```kotlin
sealed class Progress<out T> {
    data class Update<T>(val value: T, val current: Int, val total: Int) : Progress<T>()
    data class Done<T>(val result: T) : Progress<T>()
    data class Error(val error: DomainError) : Progress<Nothing>()
}

fun scanResources(sourceId: String): Flow<Progress<ScanResult>> = flow {
    val entries = fileSource.listDirectory("")
    val failures = mutableListOf<Pair<String, DomainError>>()
    entries.forEachIndexed { index, entry ->
        try {
            processEntry(entry)
            emit(Progress.Update(entry, index + 1, entries.size))
        } catch (e: Exception) {
            failures.add(entry.name to DomainError.MediaLoadError(MediaType.IMAGE, e.message ?: "", e))
        }
    }
    emit(Progress.Done(ScanResult(entries.size, failures.size, failures)))
}.flowOn(Dispatchers.IO)
```

## 7. 与 PRD 错误场景的映射

| PRD 场景 | DomainError | UI 表现 |
|----------|-------------|---------|
| 源不可达 | SourceUnreachableError | 资源置灰 + 标记 |
| 缩略图加载失败 | MediaLoadError(IMAGE) | 类型占位图 |
| 标签名校验 | ValidationError | Snackbar / 行内红字 |
| SMB 连接失败 | SourceAuthError / SourceUnreachableError | Snackbar + 源标记不可用 |
| 超大图片 OOM | MediaLoadError(IMAGE) | 加载失败占位 + 重试 |
| SMB 视频断流 | MediaStreamError(VIDEO) | 视频页失败 + 重试 |
| 加密 PDF | MediaEncryptedError(PDF) | 提示加密暂不支持 |
