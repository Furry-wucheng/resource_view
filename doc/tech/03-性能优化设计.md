## 1. 概述

本文档针对 PRD 中识别的性能敏感场景，设计系统化优化策略。核心原则：**先消峰（减少不必要的 IO/网络/解码），再并行（协程/Isolate），最后缓存（多级 LRU）。**

## 3. 首页缩略图网格（10000+ 资源）

### 3.1 虚拟滚动 — LazyVerticalGrid

Compose `LazyVerticalGrid` 天然虚拟化：仅构建可见区 + `beyondBounds` 内的 Composable，滚出即销毁。

```kotlin
LazyVerticalGrid(

    contentPadding = PaddingValues(8.dp),
    horizontalArrangement = Arrangement.spacedBy(8.dp),
    verticalArrangement = Arrangement.spacedBy(8.dp),

    items(
        items = resources,
        key = { it.id },
```

| 配置项 | 值 | 说明 |

### 3.4 请求取消与优先级

Coil 的请求生命周期与 Compose 组合树绑定：`AsyncImage` 离开 composition 时自动取消对应 `ImageRequest`。

**导航场景**（SMB 敏感）：
```
进入目录 A → LazyVerticalGrid 构建可见项 → Coil 开始加载缩略图
  │
  ├─ 用户点击子目录 B
  │     → 目录 A 的 Composable 全部 dispose → Coil 取消 A 的所有未完成请求
  │     → 目录 B 的 Composable 进入 composition → 即刻开始加载 B 的缩略图
  │
  └─ 用户点击文件进入查看器
        → 文件浏览器 dispose → 所有缩略图请求取消
        → 查看器预加载窗口接管 SMB 连接
```

**行为保证**：
- 离开的目录中还未发出的请求：不会发出，不浪费网络带宽
- SMB 已发出的 TCP 包：无法撤回，Coil 收到响应后丢弃
- 新目录的请求：立即开始，无排队，获得 smbj 连接最高优先级
- 同目录内滚动：`LazyVerticalGrid` 的 `beyondBounds` 控制预加载范围，滚出即取消

无需手写请求队列或优先级调度。Compose 声明式生命周期即是取消信号。

## 6. 缩略图磁盘缓存（LRU）

### 6.1 策略

- **淘汰策略**：LRU（最近最少使用）
- **默认容量**：500MB
- **可调档位**：500M / 1000M / 1500M / 2000M / 自定义
- **最小容量**：500MB
- **手动清理**：二次确认 → 清空所有缩略图文件

### 6.2 实现

直接使用 Coil 内置 DiskCache，不手写 LRU：

```kotlin
// Application 中统一配置（与 §3.3 的 ImageLoader 共用）
val imageLoader = ImageLoader.Builder(context)
    .diskCache {
        DiskCache.Builder()
            .directory(context.cacheDir.resolve("thumbnails"))
            .maxSizeBytes(cacheLimitBytes)  // 默认 500MB，用户在设置页调节
            .build()
    }
    .build()

// 生成缩略图时直接用 Coil 请求写入磁盘
suspend fun generateThumbnail(resourceId: String, imageData: Any): File? {
    val request = ImageRequest.Builder(context)
        .data(imageData)                   // File / Uri / ByteArray / InputStream
        .size(180, 270)                    // 按显示尺寸解码并缓存
        .build()
    val result = imageLoader.execute(request)
    return (result.drawable as? BitmapDrawable)?.let { drawable ->
        val file = File(thumbDir, "thumb_$resourceId.jpg")
        file.outputStream().use { drawable.bitmap.compress(JPEG, 85, it) }
        file
    }
}

// 加载缩略图：直接走 Coil ImageLoader 的磁盘缓存
// AsyncImage(model = File("thumb_$resourceId.jpg")) 会自动命中 DiskCache
```

Coil DiskCache 优势：LRU 淘汰、O(1) 查询、线程安全、自动统计。

## 7. SMB 视频播放与 DataSource

### 7.1 自定义 ExoPlayer DataSource

ExoPlayer 的 `DataSource` 接口是同步的，在其 I/O 线程上直接使用 smbj 的**同步阻塞 API**，避免 `runBlocking` 导致线程池饥饿：

```kotlin
class SmbDataSource(
    private val smbFile: com.hierynomus.smbj.share.File,  // smbj 同步文件句柄
    private val fileSize: Long,
) : DataSource {
    private var inputStream: InputStream? = null
    private var position: Long = 0
    private var bytesRemaining: Long = 0

    override fun open(dataSpec: DataSpec): Long {
        position = dataSpec.position
        // smbj File 支持 seek 读取
        inputStream = smbFile.getInputStream()
        inputStream?.skip(position)
        bytesRemaining = if (dataSpec.length != C.LENGTH_UNSET.toLong())
            dataSpec.length else fileSize - position
        return bytesRemaining
    }

    override fun read(buffer: ByteArray, offset: Int, length: Int): Int {
        val stream = inputStream ?: return C.RESULT_END_OF_INPUT
        val bytesToRead = minOf(length.toLong(), bytesRemaining).toInt()
        if (bytesToRead <= 0) return C.RESULT_END_OF_INPUT
        val bytesRead = stream.read(buffer, offset, bytesToRead)
        if (bytesRead > 0) {
            position += bytesRead
            bytesRemaining -= bytesRead
        }
        return if (bytesRead >= 0) bytesRead else C.RESULT_END_OF_INPUT
    }

    override fun close() { inputStream?.close() }
    override fun getUri(): Uri? = null
}

// DataSource.Factory
class SmbDataSourceFactory(
    private val smbShare: com.hierynomus.smbj.share.DiskShare,
    private val relativePath: String,
    private val fileSize: Long,
) : DataSource.Factory {
    override fun createDataSource(): DataSource {
        val smbFile = smbShare.openFile(relativePath, EnumSet.of(AccessMask.GENERIC_READ), null, null, SMB2ShareAccess.ALL, SMB2CreateDisposition.FILE_OPEN, null)
        return SmbDataSource(smbFile, fileSize)
    }
}
```

### 7.2 视频缓冲策略

| 层 | 策略 |
|----|------|
| ExoPlayer 内部缓冲 | 由播放器管理（默认 50s） |
