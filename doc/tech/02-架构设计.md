## 1. 架构总览

采用 Android 官方推荐的**分层 MVVM + Repository 模式**，配合 Hilt 依赖注入：

```
┌─────────────────────────────────────────────────┐
│  UI 层 (Presentation)                            │
│  Compose Screens ←→ ViewModels (StateFlow)       │
├─────────────────────────────────────────────────┤
│  Domain 层                                       │
│  Models (data class) + Use Cases                 │
├─────────────────────────────────────────────────┤
│  Data 层                                         │
│  Repositories (单一真相源) ← DataSources (外部 API) │
└─────────────────────────────────────────────────┘
```

**原则**：

- UI 层只做渲染和路由，所有状态从 ViewModel 通过 `StateFlow` 暴露
- ViewModel 不持有 `Context`/`Activity` 引用
- DataSource 无状态，封装外部 API（数据库、SMB、PDF、压缩包等）
- Repository 消费一个或多个 DataSource，转换为 Domain Model
- 复杂业务逻辑（跨仓库协作、资源扫描、组织模式判定）抽取为 Use Case

## 2. 技术选型

| 维度 | 方案 | 理由 |
|------|------|------|
| UI 框架 | Jetpack Compose + Material3 | Android 官方推荐，声明式 UI |
| 状态管理 | StateFlow + collectAsState | Kotlin 协程原生，生命周期安全 |
| DI | Hilt | Android 官方推荐，深度集成 ViewModel/Navigation |
| 导航 | Navigation Compose | 声明式路由，支持 DeepLink |
| 数据库 | Room (SQLite ORM) | Android 官方 ORM，Flow 响应式查询 |
| 凭据存储 | EncryptedSharedPreferences | OS 级加密（Android Keystore） |
| 图片加载 | Coil | Kotlin 协程原生，Compose 友好 |
| 视频播放 | Media3 ExoPlayer | Android 官方播放器 |
| 异步 | Kotlin Coroutines + Flow | Android 官方异步方案 |

## 4. 关键抽象设计

### 4.1 FileSource 接口（数据源策略模式）

统一本地/SMB/未来协议的文件访问：

```kotlin
interface FileSource {
    val sourceId: String
    suspend fun listDirectory(relativePath: String): List<FileEntry>
    suspend fun stat(relativePath: String): FileEntry?
    suspend fun readFile(relativePath: String): ByteArray
    suspend fun readRange(relativePath: String, offset: Long, length: Long): ByteArray
    fun openInputStream(relativePath: String): InputStream
    suspend fun testConnection(): Boolean
    fun disconnect()
}

// 实现：
//   LocalFileSource (java.io.File)
//   SmbFileSource   (smbj)
//   FtpFileSource   (P2 扩展)
//   WebDavFileSource (P3 扩展)

// 工厂：FileSourceFactory.create(Source) → 按 type 返回对应实现
```

### 4.2 ContentProvider 接口（查看器内容抽象）

```kotlin
interface ContentProvider {
    val pageCount: Int
    /** 按目标尺寸渲染指定页，控制解码分辨率，避免 OOM */
    suspend fun loadPage(index: Int, targetWidth: Int, targetHeight: Int): Bitmap
    fun dispose()
}

// ImageFolderProvider : 图片文件夹按文件名排序，Coil 按尺寸解码
// PdfContentProvider  : Android PdfRenderer 逐页渲染
// ArchiveContentProvider: 压缩包内图片列表
```

### 4.3 ViewerItem 联合类型

图片/PDF/视频在同一 ViewerScreen 内混合浏览：

```kotlin
sealed class ViewerItem {
```

## 5. 导航设计（Navigation Compose）

```kotlin
// 路由定义
sealed class Screen(val route: String) {
    data object Home : Screen("home")
    data object Sources : Screen("sources")
    data object SourceBrowser : Screen("sources/{sourceId}/browser") {
        fun createRoute(sourceId: String) = "sources/$sourceId/browser"
    }
    data object Settings : Screen("settings")
    data object Viewer : Screen("viewer/{resourceId}") {
        fun createRoute(resourceId: String) = "viewer/$resourceId"
    }
    data object FileViewer : Screen("viewer/file/{sourceId}/{path}") {
        fun createRoute(sourceId: String, path: String) = "viewer/file/$sourceId/$path"
    }
    data object TagManager : Screen("tags/manager")
}

// NavHost
NavHost(navController, startDestination = Screen.Home.route) {
    // 底部 Tab
    composable(Screen.Home.route) { HomeScreen(...) }
    composable(Screen.Sources.route) { SourceListScreen(...) }
    composable(Screen.Settings.route) { SettingsScreen(...) }

    // 全屏页面
    composable(Screen.Viewer.route) { backStackEntry ->
        val resourceId = backStackEntry.arguments?.getString("resourceId")
        ViewerScreen(resourceId = resourceId!!, ...)
    }
    composable(Screen.TagManager.route) { TagManagerScreen(...) }
    composable(Screen.SourceBrowser.route) { ... }
}
```

**要点**：
- 底部导航栏使用 `NavigationBar` + `NavHost` 实现 Tab 切换状态保持
- 查看器页面使用独立 `NavHost` 或 `fullscreen` dialog 覆盖底部导航
- ResourcePicker / 标签编辑作为 `Dialog` 弹窗

## 7. 数据流示例

### 7.1 首页标签筛选

```
用户点击标签 → HomeViewModel.selectTag(tagId)
  → filterUseCase.execute(selectedTagIds)
    → TagRepository → Room INTERSECT 查询 (Flow)
  → StateFlow<List<Resource>> 发射
  → Compose collectAsState() → LazyVerticalGrid 重组
```

### 7.2 SMB 文件浏览

```
FileBrowserViewModel.openDirectory(path)
  → filesystemRepository.listDirectory(sourceId, path)
    → FileSourceFactory.create(source) → SmbFileSource
    → withContext(Dispatchers.IO) { smbjClient.list(path) }
  → StateFlow<List<FileEntry>> 发射
  → Compose 列表/网格渲染
```

### 7.3 进入资源查看器

```
点击资源 → navController.navigate(Screen.Viewer.createRoute(resourceId))
  → ViewerViewModel.initialize(resourceId)
    → resourceRepository.getResource(id) → Resource
    → FileSourceFactory.create(source) → FileSource
    → OrganizationStrategyFactory.create(resource.organizationMode)
    → strategy.getContents(resource, fileSource) → List<FileEntry>
    → 转换为 List<ViewerItem>
  → HorizontalPager + LazyRow 渲染
    - ImagePage → Coil AsyncImage (ByteArray) 或 ContentProvider.loadPage
    - Video → ExoPlayer + MediaItem
```

## 8. Android 特有关注点

| 关注点 | 方案 |
|--------|------|
| Scoped Storage (API 30+) | 使用 SAF `DocumentFile` 访问外部存储；应用内部存储直接用 `context.filesDir` |
| 权限管理 | `READ_EXTERNAL_STORAGE` (API < 33) / `READ_MEDIA_IMAGES` (API 33+) |
| 后台扫描 | `WorkManager` 或 `CoroutineScope` + `Dispatchers.IO` |
| 进程保活 | 不需要，扫描任务通过 WorkManager 保证完成 |
| 折叠屏适配 | `WindowSizeClass` 自适应布局（手机/平板/折叠屏） |
| 深色模式 | `isSystemInDarkTheme()` + `MaterialTheme` 动态切换 |
| 返回手势 | `BackHandler` Compose 组件 |
