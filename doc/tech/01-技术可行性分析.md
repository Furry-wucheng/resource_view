## 3. 逐项详细分析

### 3.1 SMB 网络共享 — `smbj`

> 对应 PRD `@prd/03-数据源管理.md`

| 属性 | 详情 |
|------|------|
| 库 | smbj (hierynomus/smbj) |
| 版本 | 0.13.0 |
| 协议 | SMB 2.0 ~ 3.1.1（含签名/加密） |
| 平台 | JVM / Android |
| 许可证 | Apache 2.0 |
| 活跃度 | GitHub 1.8k+ stars，持续维护 |

**PRD 需求覆盖**：

| PRD 需求 | API 映射 |
|----------|---------|
| 添加 SMB 源（地址+凭据+测试连接） | `SmbClient.connect(host, port)` + `AuthenticationContext(username, password, domain)` |
| 文件浏览器目录列表 | `SmbClient.listShare()` / `DiskEntry.list()` |
| 文件元数据 | `FileStandardInformation` / `FileAllInformation` |
| 连接失败检测 | `SMBApiException` + 错误码枚举 |
| 连接恢复 | 手动重连（smbj 无自动重连池） |

**备选方案**：`jcifs-ng`（jcifs 的 SMBv2/v3 分支），不依赖 BouncyCastle，更轻量。API 更底层，配置更复杂。smbj API 更现代化，推荐。

**Android 注意**：smbj 依赖 BouncyCastle，在 Android 上需要排除冲突或使用 `bcprov-jdk18on` 的 Android 兼容版本。备选方案 `jcifs-ng` 无此依赖。

### 3.2 PDF 渲染

> 对应 PRD `@prd/04-资源查看器.md`

#### 方案 A：`pdfium-android`（推荐）

| 属性 | 详情 |
|------|------|
| 库 | `pdfium-android` (nicklausw/pdfium-android) |
| 引擎 | PDFium（Chrome 同款） |
| 平台 | Android (arm64-v8a, armeabi-v7a, x86_64) |
| 许可证 | Apache 2.0 |

**优点**：成熟稳定，支持加密 PDF，可渲染指定页为 Bitmap。

```kotlin
// 渲染单页为 Bitmap
val pdfiumCore = PdfiumCore(context)
val pdfDocument = pdfiumCore.newDocument(fd)
pdfiumCore.openPage(pdfDocument, pageIndex)
val bitmap = Bitmap.createBitmap(width, height, Bitmap.Config.ARGB_8888)
pdfiumCore.renderPageBitmap(pdfDocument, bitmap, pageIndex, 0, 0, width, height)
```

#### 方案 B：Android 内置 PdfRenderer（推荐默认方案）

| 属性 | 详情 |
|------|------|
| API | `android.graphics.pdf.PdfRenderer` |
| 最低 SDK | API 21 |
| 引擎 | 系统内置 |

**优点**：零依赖，包体小，系统级性能优化。**缺点**：不支持加密 PDF（P2 处理）。

**推荐**：默认使用 Android PdfRenderer 渲染 PDF 页为 Bitmap。加密 PDF 场景降级到 pdfium-android（按需动态集成）。

### 3.3 压缩包阅读

> 对应 PRD `@prd/04-资源查看器.md` §4.5

#### ZIP — `java.util.zip` (内置) / `zip4j`（加密场景）

| 属性 | 详情 |
|------|------|
| 库 | `java.util.zip.ZipInputStream`（JDK 内置） |
| 支持 | 标准 ZIP、无加密 |
| 许可证 | — |

基础 ZIP 直接使用 JDK 内置 `ZipInputStream`。加密 ZIP 使用 `zip4j`。

```kotlin
// 基础 ZIP（无加密）：直接用 JDK 内置 API
ZipInputStream(FileInputStream(filePath)).use { zis ->
    var entry = zis.nextEntry
    while (entry != null) {
        if (!entry.isDirectory) { /* 读取 entry */ }
        entry = zis.nextEntry
    }
}

// 加密 ZIP：使用 zip4j
val zipFile = ZipFile(filePath)
if (zipFile.isEncrypted) zipFile.setPassword(password)
val header = zipFile.getFileHeader("image.jpg")
val inputStream = zipFile.getInputStream(header)
```

#### RAR — Apache Commons Compress ✅

| 属性 | 详情 |
|------|------|
| 库 | `commons-compress` |
| 版本 | 1.26.0 |
| 支持 | RAR 4/5 |
| 许可证 | Apache 2.0 |

#### 7z — Apache Commons Compress ⚠️

7z 容器解析支持有限。**P2 再实现，优先 ZIP。**

### 3.4 视频播放 — Media3 (ExoPlayer)

> 对应 PRD `@prd/04-资源查看器.md` §4.4

| 属性 | 详情 |
|------|------|
| 库 | `androidx.media3:media3-exoplayer` |
| 版本 | 1.3.x |
| 平台 | Android |
| 许可证 | Apache 2.0 |

**PRD 需求覆盖**：

| PRD 需求 | 实现方式 |
|----------|---------|
| 单击显隐工具栏 | `Compose` + `Modifier.clickable` + 状态控制 |
| 双击暂停/播放 | `detectTapGestures(onDoubleTap)` |
| 长按倍速 | `player.setPlaybackParameters(PlaybackParameters(speed))` + 自实现手势 |
| 进度条 | Media3 `PlayerControlView` 自定义 或 自定义 Compose 组件 |
| 键盘快捷件 | `Modifier.onKeyEvent` |
| SMP 视频 | 自定义 `DataSource.Factory` 桥接 SMB 文件流 |

**SMB 视频播放方案**：
- 本地视频：直接 `MediaItem.fromUri(localPath)`
- SMB 视频：自定义 `SmbDataSource extends DataSource`，将 ExoPlayer 的读取请求桥接到 smbj 的文件流
- smbj 支持 `seek()` 随机读取，天然满足拖动进度条需求
- 禁止将 `smb://host/share/file.mp4` 直接交给 ExoPlayer（绕过凭据管理）

## 5. 风险矩阵

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|---------|
| smbj BouncyCastle Android 冲突 | 🟡 中 | SMB 功能不可用 | 排除冲突依赖或使用 Android 兼容 BC |
| pdfium-android 无活跃 fork | 🟡 中 | PDF 渲染不可用 | 降级到 Android PdfRenderer；或使用 AndroidPdfViewer 库 |
| 7z 容器解析不完整 | 🔴 高 | 7z 无法阅读 | 不列入 MVP；后续评估外部 CLI |
| ExoPlayer + SMB DataSource 复杂度 | 🟡 中 | SMB 视频体验差 | 自定义 DataSource 优先实现 Range 读取 |
| 大目录文件浏览卡顿 | 🟡 中 | UI 体验差 | Paging 3 分页加载 + 虚拟滚动 |

## 6. 开发阶段建议

| 阶段 | 内容 | 验证的核心依赖 |
|------|------|--------------|
| **P0 基础** | 数据模型 + Room + 本地源 + 图片查看器 + 标签 CRUD | Room, Coil, Hilt |
| **P0 核心** | SMB 源 + 文件浏览器 + 缩略图网格首页 + 筛选栏 + 视频播放 + 画廊模式 | smbj, Media3, Coil |
| **P1 增强** | PDF 查看器 + 章节/平铺网格模式 + 拆分资源 + ResourcePicker | pdfium-android |
| **P2 扩展** | 压缩包阅读 + 自动同步 | zip4j, commons-compress |

## 7. 架构约束

1. Source 类型（local/smb/ftp/webdav）使用**接口抽象**，为 FTP/WebDAV 预留扩展点
2. ViewerPage 需区分两种进入上下文：首页（遵守组织模式）vs 文件浏览器（平级文件滑动）
3. SMB 密码只在 `EncryptedSharedPreferences` 中存储，Room 不存明文/密文密码
4. 源不可达时 Resource.isAvailable = false（不删除，保留标签关联）
5. 缩略图缓存独立目录，LRU 淘汰，不污染数据库
6. 文件浏览器排序独立于每级目录，默认"文件夹优先+文件名升序"
7. Android 特有约束：Scoped Storage 适配（API 30+），SAF 文件访问

---

## 8. 结论

**整体技术可行性：高。** Android 原生生态对所有核心需求均有成熟方案支持。Room + Hilt + Compose + Media3 是 Android 官方推荐的标准技术栈。SMB 通过 smbj 实现，PDF 通过 pdfium-android 实现，均已有生产级验证。项目可以启动开发。
